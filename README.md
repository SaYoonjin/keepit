📌 서비스 소개 — **AI 기반 짐 보관 추천 플랫폼 KEEPIT**
========================================

\*\*KEEPIT(키핏)\*\*은  
**공공데이터 기반 공식 보관소 정보**와  
\*\*사용자 제보로 확보한 주변 유휴 공간(카페, 편의점, 상점 등)\*\*을 통합하여,

사용자에게 **위치·가격·선호도 기반 맞춤형 보관소 추천**을 제공하는  
**AI 기반 짐 보관 플랫폼**입니다.

단순히 “어디에 보관함이 있나”를 알려주는 수준을 넘어서,

✔ 실시간 보관 가능 여부  
✔ 사용자 우선순위 기반 AI 추천  
✔ 신뢰도(Trust Score) 기반 정보 검증  
✔ 누구나 자신의 공간을 보관소로 등록하여 수익 창출

까지 가능한 **새로운 형태의 도시형 짐 보관 생태계**를 구축했습니다.

* * *

🎯 개발 목표 & 문제 해결
================

### 🔸 해결하고자 한 문제

*   주변에 **즉시 사용 가능한 보관소**를 찾기 어려움
    
*   기존 지도 정보의 **실시간성 부족·부정확성**
    
*   공식 보관소만 사용 가능 → **잠재 보관 공간 미활용**
    
*   여행객·통근자·배달 기사 등 **일상 이동 불편**
    

### 🔸 KEEPIT의 해결 전략

*   **공공데이터 + 사용자 제보** 기반 보관소 통합 조회
    
*   Google Places 기반 **실시간 위치·장소 정보 자동완성**
    
*   **Weighted Hybrid Ranking AI 추천 알고리즘**
    
*   사용자 후기 기반 **신뢰도(Trust Score) 시스템**
    
*   누구나 유휴 공간을 등록하여 \*\*보관소 제공자(Host)\*\*로 참여

🧩 팀 역할 — 양박신사
==============

👨‍💻 **Backend 1 (Spring Boot — 사용자/보관소 핵심 기능)**
-------------------------------------------------

*   Spring Boot 기반 REST API 설계 및 개발
    
*   사용자 인증, 보관소 CRUD, 마이페이지 기능 구현
    
*   MySQL 기반 데이터베이스 모델링
    
*   보관소 등록/수정/삭제 로직, 유저 제보 보관소 관리
    
*   게시/비게시 및 상태 값 관리 등 서비스 도메인 로직 구현
    

👨‍💻 **Backend 2 (데이터·지도·위치 기반 기능)**
-------------------------------------

*   행정안전부·KRDI 공영 물품보관함 실시간 API 수집
    
*   Google Maps SDK & Places API 연동
    
*   공공데이터·지도 정보 정제 및 통합 저장 로직 개발
    
*   반경 기반 거리 계산, 지도 표시용 데이터 가공
    
*   사용자 위치 기반 보관소 조회 알고리즘 구현
    

🎨 **Frontend (React Native)**
------------------------------

*   모바일 UI/UX 구성 및 전체 화면 개발
    
*   지도 조회 화면, 보관소 상세 화면, 등록 페이지 제작
    
*   로그인/회원가입/마이페이지 구현
    
*   API 연동 및 상태 관리
    
*   사용자 친화적 인터랙션(UI Animation, Bottom Sheet 등) 적용
    

🤖 **AI (추천·QA·모델링)**
---------------------

*   Weighted Hybrid Ranking 기반 추천 알고리즘 설계
    
*   사용자 우선순위 설문 기반 가중치 계산 모델 개발
    
*   LightGBM/XGBoost 기반 선호도 예측 모델 구조 설계
    
*   Trust Score 모델링 및 위험도 필터링 로직 구축
    
*   Qwen2.5 기반 챗봇 QA 및 보관소 추천 대화 흐름 구성

🧠 KEEPIT AI 추천 방식 (핵심 차별점)
===========================

### ✔ 입력 데이터

*   사용자 위치(GPS)
    
*   사용자 우선순위 설문 (거리·가격·운영시간·크기 등)
    
*   보관소 속성 (실시간 상태, 가격 정보, 신뢰도 점수 등)
    

### ✔ 알고리즘

*   각 속성에 우선순위 가중치를 곱해 점수 산출
    
*   가격/거리/상태/혼잡도에 따라 페널티 또는 보너스 지급
    
*   최종 적합도 Score 기반 상위 N개 추천
    

### ✔ QA 챗봇

*   “역 근처에 큰 짐 맡길 곳 있어요?” → 조건 기반 필터링
    
*   “새벽에 이용 가능한 곳만 보여줘” → 운영시간 필터 적용
    
*   보관소 상세로 바로 연결
